## 一、数据集概述

## 二、数据格式

### 2.1 JSONL 格式

每行一条 JSON 记录：

```json
{
    "instruction": "你是一个数据分类分级助手。请根据字段名、行业和样本值判断字段属于哪一类。\n\n行业：金融\n字段名：客户年龄\n样本值示例：25, 35, 45, 55, 65 ... (共62个)\n\n请从以下分类标签中选择最合适的一个（只输出标签路径）：\n...",
    "input": "",
    "output": "身份类/人口统计"
}
```

### 2.2 字段说明

| 字段 | 说明 |
|-----|------|
| instruction | 包含任务描述、输入信息和候选标签的完整 Prompt |
| input | 辅助输入（本项目为空） |
| output | 期望的分类标签 |

## 五、数据标注规范

### 5.1 分类标注规则

| 规则 | 说明 | 示例 |
|-----|------|------|
| 唯一标识优先 | ID类字段优先识别 | id, 客户编号 |
| 名称优先 | 明显的名称字段 | name, 标题, title |
| 数值类型 | 数值型数据 | price, 金额, 数量 |
| 时间标记 | 日期时间格式 | date, 时间, update_time |
| 地址位置 | 地点类字段 | city, address, 地址 |
| 状态枚举 | 二元/多元状态 | status, type, 状态 |
| 描述文本 | 长文本字段 | description, text, 描述 |
| 其他字段 | 无法归类时使用 | 扩展类/其他字段 |

### 5.2 分级标注规则

| 规则 | 说明 | 示例 |
|-----|------|------|
| ID类 | 通常为公开 | id, 编号 |
| 名称类 | 通常为公开 | 名称, 标题 |
| 统计数据 | 通常为内部 | 销量, 访问量 |
| 个人信息 | 通常为敏感 | 年龄, 性别, 收入 |
| 财务数据 | 通常为敏感 | 工资, 资产 |
| 联系方式 | 通常为敏感 | 电话, 邮箱 |
| 内部代号 | 可为机密 | 内部编号 |

## 七、数据集文件

### 7.1 文件位置

```
data/sft/
├── train.jsonl      # 训练数据
└── val.jsonl       # 验证数据
```

### 7.2 加载方式

```python
import json

# 加载训练数据
with open('data/sft/train.jsonl', 'r', encoding='utf-8') as f:
    train_data = [json.loads(line) for line in f]

# 加载验证数据
with open('data/sft/val.jsonl', 'r', encoding='utf-8') as f:
    val_data = [json.loads(line) for line in f]

print(f"训练样本: {len(train_data)}")
print(f"验证样本: {len(val_data)}")
```
