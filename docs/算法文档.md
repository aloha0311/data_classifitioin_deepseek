# 数据分类分级系统 - 算法文档

## 一、算法概述

### 1.1 任务定义

本系统解决的是**数据字段分类分级**任务，属于自然语言理解（NLU）中的多分类问题。

**输入**：
- 字段名（column_name）
- 所属行业（industry）
- 样本值（sample_values）

**输出**：
- 分类标签（24类之一）
- 分级标签（4级之一）

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                     DeepSeek-7B + LoRA                     │
├─────────────────────────────────────────────────────────────┤
│  输入层      │  Embedding层  │  Transformer层  │  Output层   │
│  Token序列   │  语义向量     │  自注意力计算   │  标签预测    │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心技术算法

### 2.1 LoRA (Low-Rank Adaptation)

#### 2.1.1 原理

LoRA 的核心思想是：**冻结预训练模型的权重，只训练少量额外参数**。

对于原始权重矩阵 $W_0 \in \mathbb{R}^{d \times k}$，LoRA 添加低秩分解：

$$W = W_0 + \Delta W = W_0 + BA$$

其中：
- $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times k}$
- $r \ll \min(d, k)$（本项目 r=16）

#### 2.1.2 前向传播

```python
# 原始前向
h = W_0 x

# LoRA前向
h = W_0 x + BA x = W_0 x + B(Ax)
```

#### 2.1.3 参数效率

| 组件 | 参数量 | 占比 |
|-----|-------|------|
| 原始权重 | 7B | 99.9% |
| LoRA权重 | ~16M | 0.1% |

**计算公式**：
$$P_{lora} = 2 \times d \times r + r \times k + r + d \times r$$
- $d=4096$, $k=4096$, $r=16$
- $P_{lora} \approx 4 \times 4096 \times 16 = 262,144$ (单层)

#### 2.1.4 训练策略

- **冻结层**：原始权重 $W_0$ 保持不变
- **训练层**：只更新 $A$ 和 $B$ 矩阵
- **学习率**：通常比原模型学习率高 10-100 倍（本项目 2e-4）

### 2.2 Prompt Engineering

#### 2.2.1 Prompt 模板

```python
prompt = """你是一个数据分类分级助手。请根据字段名、行业和样本值判断字段属于哪一类。

行业：{industry}
字段名：{column_name}
样本值示例：{sample_values}

请从以下分类标签中选择最合适的一个：
{labels}

答案："""
```

#### 2.2.2 Few-shot 示例

系统支持通过示例引导模型理解任务：

```python
examples = [
    {

    },
    {
        "input": "字段名：name, 样本值：苹果, 香蕉, 橙子",

```

### 2.3 分类标签体系

#### 2.3.1 标签结构

采用层级标签结构：**大类/小类**

```
结构标识类
├── ID类/主键ID
├── 结构类/分类代码
├── 结构类/产品代码
├── 结构类/企业代码
└── 结构类/标准代码
```

#### 2.3.2 标签选择策略

模型需要从 24 个候选标签中选择最合适的一个。训练时使用交叉熵损失：

$$\mathcal{L} = -\sum_{i} y_i \log(\hat{y}_i)$$

其中 $\hat{y}_i$ 是模型预测的概率分布。

## 三、统计特征提取算法

### 3.1 数据类型推断

```python
def infer_data_type(values):
    # 检查数值型
    if numeric_ratio > 0.8:
        return "numeric"
    # 检查日期型
    if date_pattern_match > 0.8:
        return "date"
    # 检查文本长度
    if avg_length < 50:
        return "short_text"
    return "long_text"
```

### 3.2 字符特征提取

| 特征 | 计算方式 | 作用 |
|-----|---------|------|
| digit_ratio | 数字字符/总字符 | 识别编码类字段 |
| chinese_ratio | 中文字符/总字符 | 识别中文命名字段 |
| letter_ratio | 英文字母/总字符 | 识别英文缩写字段 |
| special_ratio | 特殊字符/总字符 | 识别分隔符字段 |

### 3.3 数值统计特征

| 特征 | 公式 | 作用 |
|-----|------|------|
| min | $\min(X)$ | 值域下限 |
| max | $\max(X)$ | 值域上限 |
| mean | $\frac{1}{n}\sum x_i$ | 平均水平 |
| std | $\sqrt{\frac{1}{n}\sum(x_i-\mu)^2}$ | 离散程度 |
| median | 中位数 | 集中趋势 |

## 四、模型评估指标

### 4.1 分类指标

| 指标 | 公式 | 说明 |
|-----|------|------|
| Accuracy | $\frac{TP+TN}{TP+TN+FP+FN}$ | 整体准确率 |
| Precision | $\frac{TP}{TP+FP}$ | 预测为正的准确率 |
| Recall | $\frac{TP}{TP+FN}$ | 正例召回率 |
| F1 | $\frac{2 \times P \times R}{P + R}$ | 调和平均 |

### 4.2 Macro vs Weighted

- **Macro**：各类别平等计算后取平均
- **Weighted**：按各类别样本数加权平均

### 4.3 本项目评估结果

| 指标 | Macro | Weighted |
|-----|-------|---------|
| Precision | 81.54% | 89.10% |
| Recall | 79.57% | 82.69% |
| F1 | 78.44% | 84.91% |

## 五、训练优化技术

### 5.1 梯度累积

由于显存限制，使用梯度累积模拟大批量训练：

```python
# 实际批次 = per_device_batch_size × gradient_accumulation_steps
effective_batch_size = 1 × 4 = 4
```

### 5.2 混合精度训练

使用 FP16 混合精度加速训练：

```python
训练Arguments:
  fp16=True
  optim="adamw_torch"
```

### 5.3 学习率调度

采用线性 warmup + 余弦衰减：

```
lr(t) = min(lr_max × t/warmup, lr_max × 0.5 × (1 + cos(πt/T)))
```

## 六、错误分析算法

### 6.1 混淆矩阵分析

分析各类别之间的误分类模式：

| 预期 | 预测 | 次数 | 占比 |
|-----|------|------|------|
| 状态类/时间标记 | 度量类/时间度量 | 2 | 22% |
| 属性类/类别标签 | 结构类/标准代码 | 1 | 11% |
| ... | ... | ... | ... |

### 6.2 错误原因归类

1. **语义相似类混淆**：时间标记 vs 时间度量
2. **上下文缺失**：仅靠字段名难以区分
3. **标注噪声**：训练样本标注可能存在不一致
