# 项目开发过程文档

## 一、项目背景

本项目旨在利用大语言模型实现数据字段的自动分类和分级功能，解决数据治理中字段分类工作繁琐、依赖专家经验的问题。

## 二、技术方案

### 2.1 核心技术选型

| 组件 | 选择 | 原因 |
|-----|------|------|
| 基础模型 | DeepSeek-LLM-7B-Chat | 强大的中文理解和推理能力 |
| 微调方法 | LoRA | 参数效率高，训练成本低 |
| 分类体系 | 24类 + 4级 | 覆盖常见数据字段类型 |

### 2.2 技术架构

```
用户输入 → Prompt构建 → DeepSeek-7B + LoRA → 分类结果 + 分级结果
```

## 三、开发流程

### 3.1 数据准备阶段

**数据收集**
- 从 Kaggle、天池等平台收集公开数据集
- 涵盖金融、医疗、教育、商业、工业等6+行业
- 最终收集40+个数据集

**数据标注**
- 针对每个数据集的字段进行人工标注
- 标注内容：分类标签（24类）+ 分级标签（4级）
- 标注规范：按字段语义、样本值特征、数据敏感性综合判断

**数据规模**
- 训练集：420条
- 验证集：46条

### 3.2 数据转换阶段

**脚本**：`batch_convert_datasets.py`

**转换内容**
- CSV → JSONL 格式
- 自动提取字段名、样本值、所属行业
- 拼接为统一 Prompt 模板

**输出文件**
- `data/sft/train.jsonl`
- `data/sft/val.jsonl`

### 3.3 模型训练阶段

**脚本**：`train_model.py`

**训练配置**
- LoRA rank: 16
- 学习率: 2e-4
- 批次大小: 1
- 梯度累积: 4
- 训练轮数: 3

**训练过程**
1. 加载预训练模型 DeepSeek-LLM-7B-Chat
2. 应用中文 tokenizer 修复
3. 配置 LoRA 参数
4. 执行 SFT 有监督微调
5. 保存 LoRA 权重到 `outputs/finetuned/`

### 3.4 模型评估阶段

**脚本**：`evaluate_model.py`

**评估指标**
- 准确率 (Accuracy): 82.69%
- 精确率 (Macro): 81.54%
- 召回率 (Macro): 79.57%
- F1分数 (Macro): 78.44%

### 3.5 批量预测阶段

**脚本**：`predict_new_data.py`

**预测流程**
1. 加载基础模型和 LoRA 权重
2. 读取 `data/new/` 目录下的 CSV 文件
3. 逐字段进行分类分级预测
4. 输出结果到 `results/csv_predict/`

## 四、辅助模块

### 4.1 动态知识库 (`dynamic_knowledge_base.py`)

支持规则动态更新和增量学习功能。

### 4.2 自监督预训练 (`self_supervised_pretraining.py`)

增强模型对字段语义的理解能力。

### 4.3 统计特征提取 (`statistical_feature_extractor.py`)

提取字段的多维统计特征，辅助分类决策。

## 五、遇到的问题与解决方案

| 问题 | 解决方案 |
|-----|---------|
| 中文 tokenizer 分词过细 | 应用 tokenizer_fix.py 进行修复 |
| 显存不足 | 使用 LoRA + FP16 混合精度 |
| 类别不平衡 | 在训练数据中进行均衡采样 |

## 六、项目产出

| 产出物 | 位置 |
|-------|------|
| 训练数据 | `data/sft/` |
| 微调权重 | `outputs/finetuned/` |
| 评估报告 | `results/model_evaluation_report.json` |
| 预测结果 | `results/csv_predict/` |

## 七、后续优化方向

1. **数据增强**：扩充训练样本，尤其是稀有类别
2. **提示工程**：优化 Prompt 模板，提升分类准确性
3. **多任务学习**：同时预测分类和分级任务
4. **错误分析**：针对易混淆类别进行专项优化
