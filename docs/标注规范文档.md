# 数据分类分级标注规范

## 一、标准依据

本标注规范参考以下国家标准：
- **GB/T 43697-2024**：《数据安全技术 数据分类分级规则》
  - 该标准规定了数据分类分级的基本原则、方法、流程
  - 适用于政务数据、企业数据的分类分级管理

## 二、术语定义

### 2.1 数据分类 (Data Classification)
根据数据的属性、特征、用途等维度，将数据划分为不同的类别。

### 2.2 数据分级 (Data Grading)
根据数据遭到篡改、破坏、泄露或非法获取、非法利用，对国家安全、公共利益或者个人、组织合法权益造成的危害程度，将数据划分为不同的级别。

### 2.3 重要数据 (Important Data)
聚焦数据安全领域，对国家、公共利益、个人和组织权益影响较大的数据。

### 2.4 核心数据 (Core Data)
关系国家安全、国民经济命脉、重要民生、重大公共利益的数据。

## 三、数据分类体系

### 3.1 分类维度

本项目采用**语义+结构**的分类维度，综合考虑：
- 字段的业务语义（是什么数据）
- 字段的数据结构特征（数值型/文本型/日期型）
- 字段的存在形式（单一值/枚举/集合）

### 3.2 一级分类（6大类）

| 大类编号 | 大类名称 | 说明 |
|---------|---------|------|
| 1 | 结构标识类 | 用于标识和编码的数据字段 |
| 2 | 属性描述类 | 描述实体属性的文本字段 |
| 3 | 数值度量类 | 包含数值测量的数据字段 |
| 4 | 身份特征类 | 与个人身份相关的特征字段 |
| 5 | 状态标记类 | 表示状态或标记的字段 |
| 6 | 扩展数据类 | 扩展或未分类的数据字段 |

### 3.3 二级分类（24类）

#### 3.3.1 结构标识类（5个）

| 二级标签 | 定义 | 典型字段 | 标注示例 |
|---------|------|---------|---------|
| ID类/主键ID | 唯一标识记录的主键或ID字段 | customer_id, user_id, order_id | customer_id → ID类/主键ID |
| 结构类/分类代码 | 用于分类标识的编码字段 | category_code, type_code | category_code → 结构类/分类代码 |
| 结构类/产品代码 | 产品或商品相关的编码字段 | product_code, sku, item_code | product_code → 结构类/产品代码 |
| 结构类/企业代码 | 企业或组织相关的编码字段 | org_code, company_code, dept_id | dept_id → 结构类/企业代码 |
| 结构类/标准代码 | 遵循标准的代码字段 | country_code, currency_code, industry_code | country_code → 结构类/标准代码 |

#### 3.3.2 属性描述类（5个）

| 二级标签 | 定义 | 典型字段 | 标注示例 |
|---------|------|---------|---------|
| 属性类/名称标题 | 实体名称或标题字段 | name, title, username, product_name | product_name → 属性类/名称标题 |
| 属性类/类别标签 | 分类或标签信息字段 | category, tag, label, type, level | category → 属性类/类别标签 |
| 属性类/描述文本 | 详细描述内容字段 | description, remark, note, content | description → 属性类/描述文本 |
| 属性类/技能标签 | 技能或能力标识字段 | skill, ability, expertise, competency | skill → 属性类/技能标签 |
| 属性类/地址位置 | 地理位置信息字段 | address, location, city, country, region | city → 属性类/地址位置 |

#### 3.3.3 数值度量类（5个）

| 二级标签 | 定义 | 典型字段 | 标注示例 |
|---------|------|---------|---------|
| 度量类/计量数值 | 连续或离散数值测量字段 | amount, price, salary, age, weight | price → 度量类/计量数值 |
| 度量类/计数统计 | 数量统计类数值字段 | count, number, total, sum, quantity | count → 度量类/计数统计 |
| 度量类/比率比例 | 百分比或比率字段 | rate, ratio, percentage, proportion | rate → 度量类/比率比例 |
| 度量类/时间度量 | 时间相关的数值字段 | duration, interval, hours, minutes | duration → 度量类/时间度量 |
| 度量类/序号排序 | 顺序编号字段 | seq, index, rank, order, position | seq → 度量类/序号排序 |

#### 3.3.4 身份特征类（4个）

| 二级标签 | 定义 | 典型字段 | 标注示例 |
|---------|------|---------|---------|
| 身份类/人口统计 | 人口统计学特征字段 | gender, birthday, age, 婚姻状况 | age → 身份类/人口统计 |
| 身份类/联系方式 | 联系方式信息字段 | phone, email, mobile, wechat | phone → 身份类/联系方式 |
| 身份类/教育背景 | 教育相关信息字段 | education, school, major, degree | education → 身份类/教育背景 |
| 身份类/职业信息 | 职业相关特征字段 | job, occupation, position, department | occupation → 身份类/职业信息 |

#### 3.3.5 状态标记类（3个）

| 二级标签 | 定义 | 典型字段 | 标注示例 |
|---------|------|---------|---------|
| 状态类/二元标志 | 二值状态标识字段 | flag, is_deleted, is_active | is_active → 状态类/二元标志 |
| 状态类/状态枚举 | 多状态枚举值字段 | status, state, stage, phase | status → 状态类/状态枚举 |
| 状态类/时间标记 | 时间点或时间戳字段 | create_time, update_time, date, timestamp | create_time → 状态类/时间标记 |

#### 3.3.6 扩展数据类（2个）

| 二级标签 | 定义 | 典型字段 | 标注示例 |
|---------|------|---------|---------|
| 扩展类/扩展代码 | 自定义扩展编码字段 | ext_code, custom_field, extra | ext_code → 扩展类/扩展代码 |
| 扩展类/其他字段 | 其他未分类字段 | other, misc, reserved | other → 扩展类/其他字段 |

## 四、数据分级体系

### 4.1 分级原则

根据**GB/T 43697-2024**的分级原则，综合考虑：
1. 数据对国家安全、公共利益的影响程度
2. 数据遭到篡改、破坏、泄露或非法获取、非法利用的影响
3. 数据关联性（是否可与其他数据关联识别特定个人或组织）
4. 数据敏感性（是否涉及个人隐私或商业秘密）

### 4.2 四级分级标准

| 级别 | 名称 | 定义 | 典型字段 | 保护要求 |
|-----|------|------|---------|---------|
| 第1级 | 公开 | 可向公众公开的数据，泄露不会造成影响 | 产品名称、公开分类代码、序号 | 无需特殊保护 |
| 第2级 | 内部 | 仅限内部人员访问，泄露会造成轻微影响 | 订单详情、部门信息、时间标记 | 基本访问控制 |
| 第3级 | 敏感 | 泄露会造成较大影响，涉及个人/敏感信息 | 薪资、教育背景、职业信息 | 加密存储、访问审计 |
| 第4级 | 机密 | 泄露会造成严重后果，高敏个人信息 | 身份证号、银行账号、密码 | 端到端加密、多因素认证 |

### 4.3 分类-分级映射规则

根据字段所属分类的默认敏感性，建立分类与分级的映射关系：

| 分类标签 | 默认分级 | 说明 |
|---------|---------|------|
| ID类/主键ID | 第1级/公开 | 标识符本身通常不敏感 |
| 结构类/分类代码 | 第1级/公开 | 标准编码通常公开 |
| 结构类/产品代码 | 第1级/公开 | 产品标识通常公开 |
| 结构类/企业代码 | 第1级/公开 | 企业编码通常公开 |
| 结构类/标准代码 | 第1级/公开 | 标准代码通常公开 |
| 属性类/名称标题 | 第1级/公开 | 名称通常公开 |
| 属性类/类别标签 | 第1级/公开 | 分类通常公开 |
| 属性类/描述文本 | 第2级/内部 | 描述可能涉及内部信息 |
| 属性类/技能标签 | 第2级/内部 | 技能信息需保护 |
| 属性类/地址位置 | 第2级/内部 | 地址信息需适度保护 |
| 度量类/计量数值 | 第2级/内部 | 数值可能涉及敏感业务 |
| 度量类/计数统计 | 第2级/内部 | 统计信息需适度保护 |
| 度量类/比率比例 | 第2级/内部 | 比率信息需适度保护 |
| 度量类/时间度量 | 第2级/内部 | 时间度量需适度保护 |
| 度量类/序号排序 | 第1级/公开 | 序号通常公开 |
| 身份类/人口统计 | 第3级/敏感 | 人口统计涉及隐私 |
| 身份类/联系方式 | 第4级/机密 | 联系方式高度敏感 |
| 身份类/教育背景 | 第3级/敏感 | 教育信息涉及隐私 |
| 身份类/职业信息 | 第3级/敏感 | 职业信息涉及隐私 |
| 状态类/二元标志 | 第2级/内部 | 状态信息需适度保护 |
| 状态类/状态枚举 | 第2级/内部 | 状态信息需适度保护 |
| 状态类/时间标记 | 第2级/内部 | 时间标记需适度保护 |
| 扩展类/扩展代码 | 第2级/内部 | 扩展信息需适度保护 |
| 扩展类/其他字段 | 第1级/公开 | 未分类字段保守处理 |

### 4.4 分级调整规则

以下情况需要**上调**分级：
1. 字段包含可识别特定个人的信息
2. 字段涉及财务、医疗、教育等敏感领域
3. 字段组合后可能推断出敏感信息

以下情况可以**下调**分级：
1. 字段已被匿名化/脱敏处理
2. 字段为公开可获取的信息
3. 字段内容已聚合统计，无法识别个体

## 五、标注流程

### 5.1 标注步骤

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤1：理解字段语义                                          │
│   - 阅读字段名，理解业务含义                                 │
│   - 查看样本值，验证理解是否正确                             │
├─────────────────────────────────────────────────────────────┤
│ 步骤2：确定一级分类                                          │
│   - 根据字段主要特征确定大类                                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤3：确定二级分类                                          │
│   - 在大类中选择最匹配的细分类                               │
├─────────────────────────────────────────────────────────────┤
│ 步骤4：确定分级                                              │
│   - 先查映射表获取默认分级                                   │
│   - 根据业务场景和样本值调整                                 │
│   - 敏感信息（如联系方式）直接定级为第3/4级                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤5：复核验证                                              │
│   - 检查分类和分级是否一致                                   │
│   - 与相似字段对比确保标注一致                               │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 常见场景标注示例

| 场景 | 字段名示例 | 分类 | 分级 | 说明 |
|------|-----------|------|------|------|

### 5.3 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 字段名含多个语义 | 以主要语义为准，参考样本值 |
| 样本值缺失(N/A) | 根据字段名判断，保守处理 |
| 中英文混合字段 | 统一处理，语义优先 |
| 字段名过于模糊 | 结合样本值和上下文判断 |
| 行业特有字段 | 参考行业知识库或专家意见 |

## 六、标注质量控制

### 6.1 一致性检验

- 同一字段名在不同数据集中标注应一致
- 相似的字段（名）应有一致的分类分级
- 定期进行标注一致性抽检

### 6.2 准确率要求

- 分类准确率目标：≥ 85%
- 分级准确率目标：≥ 90%
- 标注员间一致性（IoU）：≥ 0.80

### 6.3 标注工具要求

- 支持批量导入CSV/Excel数据
- 支持预标注和人工修正
- 支持标注历史记录和版本管理
- 支持导出标准JSONL格式
